UIView

Overview

是所有视图的基础，定义了所有视图的通用的行为

负责在矩形区域内呈现内容，并处理它的交互

可以子类化，但当你要显示应用中常见的标签，图像，按钮和其他界面元素，请使用UIKit框架提供的视图子类，而不是尝试定义自己的视图子类

主要功能（作为应用程序与用户交互的主要方式，只是一小部分）

  • Drawing and animation

    使用UIKit或Core Graphics在矩形区域绘制内容

    视图属性可以在动画中变化

  • Layout and subview management

    可以包含多个视图

    可以直接调整子视图的大小和位置

    可以自动布局来响应视图层次结构的改变

  • Event handling

    视图是UIResponder（抽象的接收和处理事件的界面）触摸和其他类型事件的一个子类

    视图可以安装手势识别器来处理常见的手势

当子视图的可见区域超出其超视图范围时，不会发生子视图内容的剪切。使用该属性来更改该行为-clipsToBounds

每个视图的几何形状由它的定义frame和bounds属性决定

  • frame:在父视图的坐标系中定义视图的原点和尺寸
  • bounds:定义了视图内部的维度，主要用于自定义绘图

使用代码创建视图

CGRect  viewRect = CGRectMake(10, 10, 100, 100);
UIView* myView = [[UIView alloc] initWithFrame:viewRect];

添加子视图 ：addSubview:

同级的子视图相互重叠，后添加的视图优先级比较高

可以指定子视图的前后关系（the relative z-order ），通过方法

  • insertSubview:aboveSubview:
  • insertSubview:belowSubview:

可以改变已经放置的子视图关系，通过

exchangeSubviewAtIndex:withSubviewAtIndex:

使用自动布局使视图能够响应视图其它层次结构的改变

动画

主要是通过改变视图的某些属性来形成动画（指定动画开始的属性值和结束的）

  • frame
  • bounds
  • center
  • transform
  • alpha
  • backgoundColor

使用UIViewPropertyAnimator对象的处理程序块更改视图属性的值，它负责执行实际的动画。可以暂时动画，来用完成交互动画

关于线程

对于用户界面的操作必须在主线程，总是在主线程使用UIView的方法，除了创建视图时，不严格在主线程

子类化视图？

仅仅在你需要的功能基本的UIView没有提供，或标准系统视图里没有的提供的时候，才去子类化。因为子类化需要大量的工作
